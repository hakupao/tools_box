# XLSX文件制作规范

本文档说明“仕样书工作流 -> 生成 Data Set”模式下，输入 XLSX 的制作要求。

适用版本：`v2.0.0`（2026-03-04）

## 1. 文件命名规范

### 1.1 基本要求
- 文件名必须以字母开头，例如 `AB.xlsx`、`CM.xlsx`、`LB.xlsx`。
- 程序会从文件名提取域名，作为输出 CSV 的 `DOMAIN`。
- 扩展名必须为 `.xlsx`。

### 1.2 示例

```text
✅ 正确示例
AB.xlsx
CM.xlsx
LB.xlsx
AE.xlsx

❌ 错误示例
123.xlsx     # 不能以数字开头
test.xlsx    # 域名不在支持列表中
AB.xls       # 扩展名错误
```

## 2. 工作表命名规范

### 2.1 支持格式
- 标准格式（推荐）：`域名+数字`，如 `AB1`、`CM2`
- 默认格式：`Sheet+数字`，如 `Sheet1`

### 2.2 规则
- 不符合命名规范的工作表会被跳过。
- 工作表顺序会影响内部序号（`SUPPSEQ`）。

## 3. 字段要求

### 3.1 通用字段
- `STUDYID`：可由流程参数统一写入。
- `DOMAIN`：由文件名域自动填充。
- `USUBJID`：受试者唯一标识。
- `[域名]SEQ`：序号字段由程序生成。

### 3.2 处理规则
- 缺失标准字段会自动补列并填空值。
- 若存在 `SUBJID`，会自动重命名为 `USUBJID`。
- 非标准字段会保留并排在标准字段后。

## 4. 数据格式要求

### 4.1 日期
- 程序会自动识别日期字段并标准化为 `YYYY-MM-DD`。
- 建议在 Excel 中使用日期类型，避免纯文本日期。

### 4.2 数据类型
- 输出时全部按字符串写入。
- 空值写为空字符串。

### 4.3 特殊字符
- 避免制表符、换行符等控制字符。
- 含逗号内容应保证 Excel 中数据结构本身正确。

## 5. 数据内容要求

- 每行应有有效的 `USUBJID`。
- 关键业务字段应完整，避免空行或逻辑不一致数据。
- 同一概念的编码、单位、格式应保持一致。

## 6. 支持域类型

### 6.1 临床域
`AG`、`CM`、`EC`、`EX`、`ML`、`PR`、`SU`

### 6.2 事件域
`AE`、`BE`、`CE`、`DS`、`DV`、`HO`、`MH`

### 6.3 检查域
`BS`、`CP`、`CV`、`DA`、`EG`、`FT`、`LB`、`MB`、`VS`

### 6.4 其他域
`DM`、`SE`、`SM`、`CO`

## 7. 排序规则

输出会按域预定义排序键排序，通常以 `USUBJID` 为主排序，再结合日期、类别或序号字段。

示例：
- AE：`USUBJID -> SUPPSEQ -> AESTDTC -> AEENDY -> AEDUR`
- CM：`USUBJID -> SUPPSEQ -> EPOCH -> CMCAT -> CMSCAT -> CMSTDTC`

## 8. 最佳实践

1. 转换前先做字段完整性检查。
2. 保持同类字段格式一致（日期、编码、单位）。
3. 建议保留原始文件备份。
4. 转换后抽样复核关键字段与记录数。

## 9. 常见问题

### 9.1 工作表被跳过
- 原因：命名不符合规范。
- 处理：改为 `域名+数字` 或 `Sheet+数字`。

### 9.2 日期异常
- 原因：原始单元格为文本。
- 处理：先在 Excel 内改为日期格式。

### 9.3 字段为空
- 原因：原数据未提供该标准字段。
- 处理：属于预期行为，程序会补列但不填默认业务值。

## 10. 输出文件说明

- 输出格式：CSV（UTF-8）
- 命名：与原 XLSX 同名，扩展名改为 `.csv`
- 内容：工作表数据合并后按标准字段顺序输出
