### 分层边界

- `src/gui/widgets/*_page.py`：界面展示、参数采集、状态反馈。
- `src/utils/*_service.py`：业务规则、文件处理、转换逻辑。
- `src/gui/qt_common.py`：统一弹窗、文件选择、文件列表控件。
- `src/gui/main_window.py`：工具注册、主页卡片、懒加载页面切换。

## 开发环境

### 要求

- Python `3.11`（当前环境验证版本）
- Windows 10/11

### 初始化

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
python src/main.py
```

## 开发约定

### 命名与组织

- 页面类命名：`<ToolName>Page`
- 服务类命名：`<ToolName>Service`
- 页面与服务保持一一对应；跨页复用逻辑放服务层或 `qt_common.py`

### 编码与数据

- CSV/Excel 读取优先 `dtype=str`，通常 `na_filter=False`
- CSV 输出统一 `utf-8-sig`
- 不改变既有分隔符/编码行为，除非需求明确要求

### 交互与异常

- 页面层统一捕获动作边界异常，使用 `show_info/show_warning/show_error`
- 文件对话框统一走 `qt_common.py` 封装

## 添加新工具流程

1. 在 `src/utils/` 新建 `*_service.py`，先实现可测试的业务接口。
2. 在 `src/gui/widgets/` 新建 `*_page.py`，只保留页面态逻辑。
3. 在 `src/gui/main_window.py` 的 `TOOL_PAGE_REGISTRY` 注册。
4. 同步更新：`README.md`、`docs/用户手册.md`、`docs/API文档.md`、`CHANGELOG.md`。

## 打包发布

### 推荐链路（onedir + installer）

```powershell
powershell -ExecutionPolicy Bypass -File .\packaging\scripts\build_onedir.ps1 -Clean
powershell -ExecutionPolicy Bypass -File .\packaging\scripts\build_installer.ps1 -Clean
```

### 启动性能对比

```powershell
powershell -ExecutionPolicy Bypass -File .\packaging\scripts\measure_startup.ps1
```

### 兼容链路（one-file）

```powershell
python -m PyInstaller tools_box.spec --clean
```
